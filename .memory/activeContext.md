# **הקשר רגעי ופעיל (Sliding Window Temporal Memory)**

## **מוקד הפעילות הנוכחי (Current Focus)**

המשימה הנוכחית: ביצוע דיבוג ואופטימיזציה לחווית הגלילה, פתרון כפילויות מסכי סיום ויצירת חווית מעבר חלקה בין נושאי לימוד.

## **חלון עשרת האירועים האחרונים (Sliding Window)**

1. **שינוי ארכיטקטורת רינדור נושאים ב-HomeClient:** העברת המערכת מטעינה רציפה של כל 30+ הנושאים, לרינדור לפי `activeLessonId`. מסך הפתיחה ("ברוכים הבאים") הוצמד תמיד לראש הדף כריל קבוע.
2. **שדרוג מסך הגיבור (Hero Section):** הוטמעו בתוכו תיבת החיפוש ומגירת הנושאים במלואה (כך שהמשתמש יכול לראות את הקטגוריות והנושאים היישר ממסך הפתיחה).
3. **יצירת כרטיסיית TopicIntroCard:** פותחה קומפוננטה חדשה המייצרת ריל פתיחה מונפש וחגיגי (בגווני ורוד וסגול) עבור כל נושא חדש. ריל זה מוזרק אוטומטית לפני הרצפים בכל `lesson` ב-`page.tsx`.
4. **שדרוג SequenceEndCard:** נוסף כפתור "בחר נושא אחר" הפותח את מגירת הנושאים מתוך מסך הסיום של נושא.
5. **בדיקת תקינות מערכת (npm run build):** וידוא תקינות בסיסית - הפרויקט קומפל בהצלחה.
6. **תיקון באג מסכי סיום כפולים:** הוספת סינון ב-`page.tsx` (`groupBitesBySequence`) למחיקת `sequence_end_card` המופיעים בטעות ב-JSON המקורי, תוך השענות מוחלטת על ה-Bridge Card.
7. **שדרוג חווית מעבר בין נושאים (Topic Transition):** ב-`HomeClient.tsx` יצרנו שכבת Overlay בצבעי תכלת-כחול באמצעות Framer Motion. כעת בעת החלפת נושא, המערכת מבצעת Fade In של האוברליי, מבצעת קפיצת גלילה אוטומטית וללא אנימציה (Instant scroll) אל ראש הדף, ולאחר מכן Fade Out שמגלה את הנושא החדש.
8. **אופטימיזציית ביצועי גלילה ו-Framer Motion:** הוסרו התנגשויות CSS (`scroll-smooth`) שהפריעו ל-`snap-mandatory`. יתרה מכך, בסריקה רוחבית תוקנו 21 מופעים של `viewport={{ once: false }}` בכל הבלוקים (Principle, Clinical, Compare) ל-`once: true` כדי למנוע טעינה מחדש של אנימציות בכל גלילה - מה שפתר לחלוטין את ה-"Stutters" ואת העומס על הדפדפן.
9. **אימות מערכת:** פקודת `npm run build` הושלמה שוב ללא שגיאות.
10. **העשרת תוכן קליני (סוכרת במים):** התוכן בקובץ `physiology_05_diabetes.json` שודרג משמעותית. נשמר הטקסט המקורי אך התווסף "רקע פיזיולוגי" מקדים (Contextual Foundation) המכיל הסברים על השפעת המים החמים והלחץ ההידרוסטטי על מחזור הדם (כרטיסיית עקרון) ועל הבדלי ניצול הגלוקוז בין מים ליבשה (כרטיסיית השוואה). בנוסף, שולבה קרוסלת דוגמאות (flashcard_carousel) לזיהוי סימני אזהרה להיפוגליקמיה במים מאחר והמים ממסכים זיעה. כרטיסיות המבחן שונו מ-`quiz_card` ל-`interactive_check` על מנת ליישר קו עם הגדרות המערכת החדשות (System Patterns). בסיום הרצף שולב `sequence_end_card` תקני. השדרוג שומר על עקרון ההפרדה המוחלטת (Decoupling) מאחר והשינויים נעשו רק במבנה הנתונים ב-JSON ולא נכתב קוד UI.

*הנחיית מערכת:* חובה לתעד כל פעולה בחלון זה באופן מפורט, אינפורמטיבי וממצה (כולל מה נעשה, סיבות להחלטות ושינויים טכניים מקיפים), ולא בשורה קצרה וכללית. תקצור ודחיסה של מידע יבוצעו **אך ורק** כאשר חלון זה מתמלא (אירוע 11 ואילך), ואז על הסוכן להעביר את האירועים הישנים לתוך `.memory/changelog.md` תוך כדי דחיסה. ה-activeContext עצמו חייב להישאר מפורט במלואו.
