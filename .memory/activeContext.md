# **הקשר רגעי ופעיל (Sliding Window Temporal Memory)**

## **מוקד הפעילות הנוכחי (Current Focus)**

המשימה הנוכחית: העשרת תוכן קליני ופירוק דחיסות קונספטואלית (Conceptual Compression) ביחידת טרשת נפוצה (MS).

## **חלון עשרת האירועים האחרונים (Sliding Window)**

1. **שינוי ארכיטקטורת רינדור נושאים ב-HomeClient:** העברת המערכת מטעינה רציפה של כל 30+ הנושאים, לרינדור לפי `activeLessonId`. מסך הפתיחה ("ברוכים הבאים") הוצמד תמיד לראש הדף כריל קבוע.
2. **שדרוג מסך הגיבור (Hero Section):** הוטמעו בתוכו תיבת החיפוש ומגירת הנושאים במלואה (כך שהמשתמש יכול לראות את הקטגוריות והנושאים היישר ממסך הפתיחה).
3. **יצירת כרטיסיית TopicIntroCard:** פותחה קומפוננטה חדשה המייצרת ריל פתיחה מונפש וחגיגי (בגווני ורוד וסגול) עבור כל נושא חדש. ריל זה מוזרק אוטומטית לפני הרצפים בכל `lesson` ב-`page.tsx`.
4. **שדרוג SequenceEndCard:** נוסף כפתור "בחר נושא אחר" הפותח את מגירת הנושאים מתוך מסך הסיום של נושא.
5. **בדיקת תקינות מערכת (npm run build):** וידוא תקינות בסיסית - הפרויקט קומפל בהצלחה.
6. **תיקון באג מסכי סיום כפולים:** הוספת סינון ב-`page.tsx` (`groupBitesBySequence`) למחיקת `sequence_end_card` המופיעים בטעות ב-JSON המקורי, תוך השענות מוחלטת על ה-Bridge Card.
7. **שדרוג חווית מעבר בין נושאים (Topic Transition):** ב-`HomeClient.tsx` יצרנו שכבת Overlay בצבעי תכלת-כחול באמצעות Framer Motion. כעת בעת החלפת נושא, המערכת מבצעת Fade In של האוברליי, מבצעת קפיצת גלילה אוטומטית וללא אנימציה (Instant scroll) אל ראש הדף, ולאחר מכן Fade Out שמגלה את הנושא החדש.
8. **אופטימיזציית ביצועי גלילה ו-Framer Motion:** הוסרו התנגשויות CSS (`scroll-smooth`) שהפריעו ל-`snap-mandatory`. יתרה מכך, בסריקה רוחבית תוקנו 21 מופעים של `viewport={{ once: false }}` בכל הבלוקים (Principle, Clinical, Compare) ל-`once: true` כדי למנוע טעינה מחדש של אנימציות בכל גלילה - מה שפתר לחלוטין את ה-"Stutters" ואת העומס על הדפדפן.
9. **אימות מערכת:** פקודת `npm run build` הושלמה שוב ללא שגיאות.
10. **העשרת תוכן קליני (MS):** שוכתב והועשר הקובץ `neuro_06_ms.json` בהתאם להנחיות ה-Clinical Content Director. הוסף רקע פתופיזיולוגי מקדים ("חוק ההקשר"), וקרוסלת הסימפטומים הדחוסה פורקה לרצפים נפרדים של תשישות וספסטיות (עיקרון -> מקרה קליני -> יישום), ללא מחיקת טקסטים קיימים ועם שימור אנימציות ליבה קיימות כנדרש מ-`systemPatterns.md`.
10. **העשרת תוכן קליני ל-DCD (Pediatrics):** בוצעה הרחבה והעמקה של קובץ `pediatrics_02_dcd.json`. יושם חוק "Contextual Foundation" על ידי הוספת ריל תיאורטי המסביר את הבסיס הנוירו-מוטורי לפני המעבר לפרקטיקה. יושם חוק "No Conceptual Compression" על ידי פירוק קרוסלת המאפיינים הדחוסה ל-3 רילסים נפרדים מסוג `clinical_case_card`, כאשר הושארה קרוסלה בסוף הרצף רק עבור דוגמאות לתרגילים מעשיים. הממשק הועשר בטקסטים (כגון "הידעת?") תוך שמירה על כללי ה-visual_trigger וההקשר של מעכב הקצב במים.

10. **העמקה והעשרת תוכן קליני (POTS):** כ-Clinical Content Director, בוצע שכתוב מקיף של הקובץ `app_build/content_data/lessons/neuro_03_pots.json`. בהתאם לעקרונות ה-Decoupling ו"חוק איסור דחיסה", התוכן פוצל ל-4 רצפים לוגיים (פיזיולוגיה, מנגנון פעולה, פרוטוקול טיפול, סיכום ומבדק). הוספו מסכי `compare_card`, `flashcard_carousel` (להצגת דוגמאות תרגילים באופן מופרד ומפורט), כרטיסי `sequence_end_card` למעברים בין פרקים, ושדות העשרה חדשים כמו `did_you_know`. כמו כן תוקן טיפוס השאלה מ-`quiz_card` ל-`interactive_check` בהתאם לסכמה הרשמית. לא הומצאו אנימציות חדשות אלא נעשה שימוש מושכל באנימציות קיימות.

*הנחיות מערכת:* חובה לתעד כל פעולה בחלון זה באופן מפורט, אינפורמטיבי וממצה (כולל מה נעשה, סיבות להחלטות ושינויים טכניים מקיפים), ולא בשורה קצרה וכללית. תקצור ודחיסה של מידע יבוצעו **אך ורק** כאשר חלון זה מתמלא (אירוע 11 ואילך), ואז על הסוכן להעביר את האירועים הישנים לתוך `.memory/changelog.md` תוך כדי דחיסה. ה-activeContext עצמו חייב להישאר מפורט במלואו.
