# **הקשר רגעי ופעיל (Sliding Window Temporal Memory)**

## **מוקד הפעילות הנוכחי (Current Focus)**

המשימה הנוכחית: פיתוח אנימציות חיות מבוססות חוקי הפיזיקה (Phase 3). המערכת עלתה לפרודקשן (Next.js + Tailwind + Framer Motion) עם 5 בלוקי למידה דינמיים הניזונים מ-JSON. שדרוג ה-`AnimationFactory` נמשך, כעת ממוקד באנימציות אינטראקטיביות וחישוב כוחות דינמיים (CG/CB). ארכיטקטורת המידע שודרגה לתמיכה ברצפי למידה (Sequences) עם כותרות קפואות.

## **חלון עשרת האירועים האחרונים (Sliding Window)**

1. הקמת מעטפת ה-Mobile-First (`ReelContainer`) ואנימציות Framer.
2. בניית 5 קומפוננטות ה-Blocks הגנריות תחת `components/blocks`.
3. פיתוח ה-`ReelRenderer` שמשמש כפקטורי להצגת הנתונים מה-JSON ל-Blocks.
4. תיקוני Build ב-Next.js (הזרקת `use client` לרכיבי UI).
5. הפעלת שרת פיתוח (`npm run dev`) עם הוכחת היתכנות מלאה למערכת הדינמית.
6. יצירת Implementation Plan לשלב 3: פיתוח אנימציות פיזיקה ליבתיות והגדרת Fallback.
7. עדכון ה-Memory Bank למעקב אחרי 18 ה-Visual Triggers הנותרים שמהווים חוב טכני לעתיד.
8. שדרוג אנימציית MetacentricTorque למערכת אינטראקטיבית המדמה קטיעות גפיים והשפעתן על מרכז הכובד והציפה.
9. שדרוג הארכיטקטורה וה-UI: תמיכה ברצפי רילסים (Sticky Sequence Titles) והבלטת תגית "דוגמה קלינית".
10. **עדכון נהלי תיעוד מערכתיים:** הטמענו נהלים חדשים ונוקשים ב-AGENTS.md וב-activeContext.md הדורשים כי כל רישום ב-activeContext יהיה מפורט, אינפורמטיבי וממצה (כולל החלטות טכניות ושינויים ארכיטקטוניים). הוגדר במפורש כי דחיסה או תקצור של רישומים אסורים במסגרת ה-activeContext, ומותרים אך ורק בעת ארכוב הרישומים הישנים לתוך קובץ ה-changelog.md. כעת האג'נטים יתעדו כל פעולה בצורה מורחבת על מנת לאפשר קונטקסט מלא לפעולות עתידיות.

*הנחיית מערכת:* חובה לתעד כל פעולה בחלון זה באופן מפורט, אינפורמטיבי וממצה (כולל מה נעשה, סיבות להחלטות ושינויים טכניים מקיפים), ולא בשורה קצרה וכללית. תקצור ודחיסה של מידע יבוצעו **אך ורק** כאשר חלון זה מתמלא (אירוע 11 ואילך), ואז על הסוכן להעביר את האירועים הישנים לתוך `.memory/changelog.md` תוך כדי דחיסה. ה-activeContext עצמו חייב להישאר מפורט במלואו.1
