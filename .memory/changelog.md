# **היסטוריית שינויים ואירועים דחוסה (Changelog)**

* **הזרקת יחידה 1:** הושלמה הזרקת התוכן הרפואי מיחידה 1 ליצירת `unit1_physics.json` תחת דרישת ההפרדה המוחלטת (Decoupling) בין UI לתוכן.
* **הזרקת יחידה 2:** הושלמה הזרקת התוכן הרפואי מיחידה 2 ליצירת `unit2_pathologies.json`.
* **בקרת איכות התחלתית (QA):** עברה בהצלחה בדיקת עמידה במגבלות מילים (עד 40) ובטיחות רפואית (MS/Diabetes).
* **בניית תשתית פרונטאנד דינמית (UI):** הוקמה מעטפת Mobile-First מלאה (ReelContainer ו-5 בלוקים גנריים) ופותח ReelRenderer לשאיבת נתונים מקבצי JSON. בוצעו תיקוני Build ב-Next.js והמערכת הועלתה לפרודקשן (npm run dev) עם הוכחת היתכנות מלאה למערכת דינמית.
* **הקמת תשתית UI ל-Mobile-First (שלבים 1-2):** פיתוח מעטפת ה-`ReelContainer` ואנימציות Framer, בניית קומפוננטות הבלוקים וה-`ReelRenderer` שמשמש כ-Factory לנתוני ה-JSON, פתרון שגיאות Build והפעלת POC מלא בסביבת פיתוח מקומית.
* **הקמת תשתית Mobile-First והוכחת היתכנות:** פיתוח רכיבי המעטפת (ReelContainer, ReelRenderer), הוספת בלוקים גנריים, תיקוני Build של Next.js (use client) והפעלת שרת פיתוח המוכיח מערכת דינמית הניזונה מ-JSON.* **שלב הוכחת היתכנות של פיתוח ה-UI (MVP):** הוקמה מעטפת Mobile-First מלאה מבוססת Next.js ו-Framer Motion, שכללה בניית 5 קומפוננטות UI בסיסיות לרינדור JSON ופקטורי (`ReelRenderer`) והרצת שרת פיתוח להוכחת הפעלתן. תוקנו בעיות Build הקשורות לרכיבי צד-לקוח.
* **יצירת Implementation Plan לשלב 3:** פיתוח אנימציות פיזיקה ליבתיות והגדרת Fallback.
* **עדכוני UI ואנימציות (שלבים מתקדמים):** שיפור משמעותי במעטפת הויזואלית, כולל אנימציות דינמיות. שדרוג MetacentricTorque למודל אינטראקטיבי לקטיעות גפיים, פתרון בעיות קריסה ו-Layout בכרטיסיות ויצירת חוק ה-Anti-Collision ב-`design_and_pedagogy_principles.md`. המערכת הורחבה לתמוך ביצירת עמודי קורסים דינמיים (`generateStaticParams`).
* **הזרקת תוכן קליני מתקדם:** מיפוי והזרקת 4 מערכי שיעור חדשים (כליות, אורתופדיה, אפילפסיה וגפיים תחתונות) לפורמט ה-JSON תחת מגבלת ה-40 מילים לכרטיס. הוסרו הגבלות והפניות מחייבות לשיטת הליוויק (Halliwick) מכללי המערכת.
* **סנכרון מערכת ו-UX:** ארכיטקטורת UX חדשה מבוססת Mobile-First (חיפוש מתקדם, עמוד קטלוג). נכתבו 4 אנימציות קליניות והוקמו עוד 3 על ידי סוכני אנימציה מותאמים אישית (`SpasticityGamma`, `PainReliefMovement`, `UhthoffMS`). קוד נבדק בעזרת `npm run build` ועלה בהצלחה ל-GitHub.
* **יצירת מרחב עבודה פדגוגי (.fromhuman):** הוקמה תשתית לניתוח שאלות מבחן וזיהוי פערי ידע.
* **בנייה פדגוגית מלאה (Holistic Pedagogy):** מיפוי פערים, הגדרת 6 תבניות פדגוגיות (Templates), יצירת חוק קרוסלות מפורטות והפעלה נרחבת של תתי-סוכנים לייצור ועיבוי עשרות קבצי JSON חדשים בכלל נושאי הקורס (אנטומיה, ילדים, נוירולוגיה).
* **QA & Governance (VIGIL):** בדיקה קפדנית של רכיבים חדשים ותיקון כיווניות (LTR) של תווית בקומפוננטת `animation_pwb_c7`.
* **פיתוח פדגוגי חלק 2:** סוכן התוכן יצר קבצי JSON חדשים (ילדים, נוירולוגיה, אורתופדיה, אנטומיה ותנועה) עם עיצוב קרוסלה.
* **הזרקת כרטיסיות בוחן:** הזרקת אובייקטי `quiz_card` עם שאלות רב-ברירה לקבצי התוכן בתיקיית `lessons`. שולבה קומפוננטת `QuizCard.tsx` למערכת.
* **עדכון קטלוג:** הוספת קטגוריות ו-30 קבצי התוכן לקטלוג, ושדרוג מגירת הנושאים (`TopicBottomSheet.tsx`) לתמיכה בגלילה מותאמת.
* **אופטימיזציית חווית משתמש ומעברים (UX & Transitions):** מעבר לרינדור לפי נושאים פעילים (`activeLessonId`), פיתוח כרטיסיית `TopicIntroCard`, ושדרוג שכבת ה-Overlay במעברים עם קפיצת גלילה אוטומטית (Instant Scroll). פתרון בעיית מסכי סיום כפולים ב-JSON המקורי, ותיקון 21 מופעים של אנימציות נטענות מחדש כדי למנוע Stutters ולשפר משמעותית את ביצועי הגלילה.
